# 장애물 회피 (Collision Avoidance) 원리 설명

## 1. 무엇을 하는 예제인가요?

카메라로 앞을 보고, "지금 이 사진에 장애물이 가까이 있는가?"를 판단하는
AI 모델을 직접 만들어서, 오토카가 장애물을 만나면 스스로 피하도록 만드는 예제입니다.

전체 흐름은 3단계입니다.

1. **데이터 수집**: 카메라로 사진을 찍어서 "장애물 있음(blocked)" / "장애물 없음(free)"으로 분류해 저장
2. **딥러닝 학습**: 모아둔 사진으로 신경망(CNN)을 훈련
3. **주행**: 학습된 모델이 실시간으로 "지금 장애물 있나 없나"를 판단 → 있으면 후진, 없으면 전진

## 2. 왜 사진을 직접 모아야 하나요?

이 AI는 "사람이 규칙을 짜서" 장애물을 피하는 게 아니라,
**많은 예시(사진)를 보여주고 스스로 패턴을 배우게 하는 방식(지도학습, Supervised Learning)**입니다.

- `free` 사진: 앞이 뚫려 있는 장면들
- `blocked` 사진: 벽, 사람, 물건 등이 가까이 있는 장면들

이 두 종류의 사진을 최대한 많이(각 300장 이상), 다양한 장소(실내/실외/타일바닥 등)에서
모아야 모델이 "진짜 장애물이 무엇인지"를 잘 배웁니다.
→ 데이터가 적거나 한 장소에서만 찍으면, 그 장소 밖에서는 잘 못 맞춥니다.
(예제 원문에서도 "새로운 조명/바닥에서는 제대로 주행하지 않는다"고 나온 이유입니다.)

## 3. 딥러닝(CNN)은 어떻게 판단하나요?

`Collision_Avoid` 클래스 내부에는 **CNN(합성곱 신경망, Convolutional Neural Network)**이 들어 있습니다.

- CNN은 사진을 작은 부분(패턴)들로 나눠서 보고, "이런 모양이 있으면 장애물일 가능성이 높다"는
  특징들을 학습 과정에서 스스로 찾아냅니다.
- 학습이 끝나면, 새로운 사진 한 장을 넣었을 때 **0~1 사이의 숫자(확률)**를 출력합니다.
  - 0에 가까움 → 장애물이 없을 확률이 높음
  - 1에 가까움 → 장애물이 있을 확률이 높음

`CA.train(times=10)`에서 `times`는 전체 데이터를 몇 번 반복해서 학습할지를 의미합니다.
(학습 중 출력되는 `accuracy`는 정확도로, 1.0에 가까울수록 잘 맞춘다는 뜻입니다.)

## 4. 학습된 모델로 어떻게 주행하나요?

```python
def drive(value):
    if value <= 0.5:
        Car.forward()   # 장애물 없음 -> 전진
    else:
        Car.backward()  # 장애물 있음 -> 후진(회피)
```

- `CA.run()`이 카메라 영상 한 장을 모델에 넣어 확률(`value`)을 계산합니다.
- `callback`으로 넘긴 `drive` 함수가 그 확률을 받아서 실제 모터(전진/후진)를 제어합니다.
- `while True`로 이 과정을 계속 반복하기 때문에, 오토카가 실시간으로 앞을 보며 피하게 됩니다.

## 5. 라이다(LiDAR)를 같이 쓰면 뭐가 달라지나요?

카메라 딥러닝 모델만 쓰면 두 가지 약점이 있습니다.

- 카메라는 "확률"로 판단하기 때문에 100% 확신은 아닙니다. 가끔 코앞에 장애물이 있어도 오판할 수 있습니다.
- 카메라는 처리 속도가 느려서(사진 → CNN 계산) 반응이 살짝 늦을 수 있습니다.

라이다는 "정면까지의 실제 거리(mm)"를 직접 재기 때문에, 정말 가까이 뭔가 있으면 카메라 판단과
상관없이 **무조건 멈추는 안전장치**로 쓰기 좋습니다. 그래서 역할을 이렇게 나눕니다.

| 센서 | 역할 |
|---|---|
| 라이다 | 정면이 설정한 안전거리(`STOP_DISTANCE`, mm)보다 가까우면 무조건 정지 (최우선 안전장치) |
| 카메라 + CNN | 라이다가 위험을 안 알렸을 때, "장애물일 확률"을 보고 전진할지 후진할지 판단 |

```python
def drive(value):
    if dist is not None and dist < STOP_DISTANCE:
        Car.stop()          # 라이다: 너무 가까우면 무조건 정지 (카메라 판단보다 우선)
        return
    if value <= 0.5:
        Car.forward()       # 카메라: 장애물 없다고 판단 -> 전진
    else:
        Car.backward()      # 카메라: 장애물 있다고 판단 -> 후진
```

라이다는 백그라운드 스레드(`lidar_update`)에서 계속 정면 ±10도 범위의 거리를 평균 내서
`front_distance` 변수에 저장해두고, 카메라의 `drive` 콜백이 매번 그 값을 확인하는 구조입니다.
(라이다는 회전하면서 데이터를 주기 때문에, 정면 근처 여러 각도의 값을 평균 내야 노이즈가 줄어듭니다.)

## 6. 파일 구성

| 파일 | 역할 |
|---|---|
| `1_데이터_수집.py` | 카메라를 켜고 GUI로 free/blocked 사진을 모음 (`collision_dataset/` 폴더 생성) |
| `2_딥러닝_학습.py` | 모은 사진으로 학습만 하고, 모델을 `collision_avoid_model.pth`로 저장 (주행 안 함) |
| `3_카메라_주행.py` | 저장된 모델을 불러와서, 학습 없이 카메라만 이용해 바로 주행 |
| `4_라이다_카메라_통합_주행.py` | 저장된 모델을 불러와서, 라이다(안전정지) + 카메라(회피 판단)를 같이 사용해 주행 |

## 7. 실행 순서 (요약)

1. **오토카(Jetson) Jupyter Notebook**에서 `1_데이터_수집.py`의 코드를 셀 단위로 실행 → 사진 데이터 모으기
2. 같은 환경에서 `2_딥러닝_학습.py` 실행 → 학습 후 `collision_avoid_model.pth`로 모델 저장
3. `3_카메라_주행.py` 실행 → 저장된 모델을 불러와서 카메라만으로 주행 확인 (다시 학습할 필요 없음)
4. `4_라이다_카메라_통합_주행.py` 실행 → 저장된 모델 + 라이다를 같이 사용해 주행
   - `STOP_DISTANCE`(기본 300mm)를 오토카 크기/속도에 맞게 조절하세요. 너무 작으면 못 멈추고 부딪히고, 너무 크면 자주 멈춥니다.
5. 잘 안 피하면 → 다양한 장소/각도에서 사진을 더 모아서 1번부터 다시 진행 (데이터가 많을수록 성능 향상)
