## 2026-07-13

**담당: 자율주행 전체 (라이다 + YOLO + OpenCV)**

### 완료
- **YOLO 객체 탐지**: 오토카(Jetson)에서 YOLOv3-tiny + TensorRT로 실시간 탐지 성공. **FPS 6.0 → 30.1**로 5배 개선
- **장애물 회피 로직**: "정지 후 생각" 방식으로 작성, 실행 확인 (`jetson_obstacle_avoidance.py`)
- **TTS 음성 알림**: 장애물 감지 시 한국어 음성 안내 연동
- **라인 트레이싱 코드 작성**: OpenCV 색상 검출 방식 (`line_tracing.py`), 장애물 회피와 통합 시 "목표 지점을 옆으로 잠깐 옮기는" 방식으로 자연스럽게 원래 라인 복귀하도록 설계
- **라이브러리 버그 수정**: `pop/Pilot.py`의 numpy import 누락 수정
- **사람 따라가기 코드 작성**: YOLO로 person 인식 → 화면 중앙 유지 + 거리 기반 전진/정지 (`jetson_follow_person.py`, 색상 보정 불필요라 라인 트레이싱 대신 우선 작성)

### 시도했으나 미완료
- **트랙 현장 라인 색상 보정**: 강한 역광(정오 햇빛)으로 흰 선과 빨간 트랙의 색 구분이 잘 안 됨. 카메라 각도 조정으로 일부 개선했으나 완전한 보정은 실패
  → **다음: 흐린 날/저녁에 재시도, 카메라 차양막(렌즈 후드) 준비**
- **사람 따라가기 실기 테스트**: 코드만 작성, 더위로 현장 테스트는 다음으로 미룸

### 역할 정리
| 센서 | 역할 |
|---|---|
| 라이다 | 360도 거리 측정, 정지/회피 결정 (메인) |
| YOLO | 물체 종류 판단 (보조, TTS 안내용) + 사람 따라가기 |
| OpenCV | 트랙 라인 중앙 유지 (라인 트레이싱, 보류 중) |

## 2026-07-14

**담당: 자율주행 전체 (라이다 + YOLO + 폰 조종 통합)**

### 완료
- **사람 따라가기 실기 디버깅**: 차가 안 움직이던 원인(`setSpeed()`가 정지 상태에서 무효였던 버그) 발견/수정, 인식 떨림으로 정지↔전진 반복되던 문제 스무딩+디바운스로 해결
- **라이다 실제 연결 + 안전 회피 통합**: 700mm 이내 장애물 감지 시 어떤 모드든 최우선으로 회피하도록 구현, 실제 연결 확인
- **폰 조종 기능**: 팀 저장소의 `app.py`+`remote.html` 가져와서 YOLO 사람 추적(AUTO)/폰 조종(MANUAL) 하나로 통합 (`phone_remote_control.py`)
- **시동/종료 버튼**: 상시 대기하는 가벼운 런처(`launcher.py`, 포트 5001)로 폰에서 tmux 켜고 끄기 가능
- **사람 놓치면 카메라 탐색**: 카메라 서보를 90(중앙)→180→0→90 순서로 천천히 왕복(20렙), 그래도 못 찾으면 AUTO 자동 종료
- **크래시 2건 해결**: 라이다를 별도 스레드로 돌리면 죽는 문제(스레드 제거로 해결), 카메라/라이다 동시 전원 사용으로 죽는 문제(연결 전 3초 대기로 해결)
- 안 쓰는 파일 정리 (WSL/Jetson 양쪽), 팀 저장소 구조 파악

### 시도했으나 미완료
- 폰으로 직접 MANUAL/AUTO 실기 조종 테스트는 방금 안정화되어 아직 충분히 못 해봄
- 카메라 각도(라인트레이싱 때 낮춘 각도) 재확인 필요

### 역할 정리
| 구성 | 역할 |
|---|---|
| 라이다 | 근접 장애물(700mm) 회피, MANUAL/AUTO 공통 최우선 안전장치 |
| YOLO | AUTO 모드에서 사람 추적, 놓치면 카메라 탐색 |
| 폰 웹뷰 | MANUAL 조종 + AUTO 전환 + 시동/종료 |
